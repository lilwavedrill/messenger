## Работа в команде

решения (выбор транспорта, схема БД, модель авторизации, договорённости

## Участник 1 — Фамилия Имя (`lilwavedrill`)

**Роль:** Backend / БД / инфраструктура

**Что сделано:**

- Первоначальное проектирование системы и схемы БД
  (`backend/schema.sql`, ER-диаграмма в `docs/er.*`).
- Настройка окружения проекта: `requirements.txt`, `.env.example`,
  `scripts/seed.py` для наполнения БД тестовыми данными.
- Ядро серверного приложения:
  - маршруты аутентификации первой итерации (`routes_auth.py`);
  - работа с чатами и участниками, роли `reader/writer/owner`
    (`routes_chats.py`);
  - маршруты сообщений, история с пагинацией, поиск
    (`routes_messages.py`);
  - WebSocket-эндпоинт и in-memory реестр подписчиков
    (`routes_ws.py`, `hub.py`).
- Статусы доставки сообщений (`sent` / `delivered` / `read`):
  таблица `message_status`, WS-события `ack` / `read`
  (`migrations/001_message_status.sql`).
- Интеграция с S3-совместимым хранилищем (MinIO):
  клиент, загрузка файлов, генерация presigned-URL
  (`app/s3.py`, `routes_attachments.py`,
  `migrations/002_attachments.sql`).
- Идемпотентность отправки сообщений через `client_msg_id`
  и уникальный ключ `(chat_id, client_msg_id)`.
- Первоначальная версия отчёта, ER-диаграмма
  (`docs/build_report.js`, `docs/er.dot`), инструкция по запуску
  в README.
- Настройка репозитория на GitHub, работа с приглашениями коллаборатора.

**Разделы отчёта:** 1 (аналитическая часть), 2.1 (архитектура),
2.2 (структура БД), приложение А (ER-диаграмма).

## Участник 2 — Фамилия Имя (`IvanMamontov1`)

**Роль:** Frontend / расширенная авторизация / тестирование

**Что сделано:**

- Полная реализация веб-клиента: одностраничное приложение без
  сборщика, современный UI (`frontend/index.html`).
  Разделение личных и групповых чатов, сворачиваемый сайдбар,
  экран профиля с аватаром, лайтбокс для просмотра картинок.
- Расширение модели авторизации: `refresh`-токены с ротацией
  и отзывом, эндпоинты `/auth/refresh` и `/auth/logout`
  (`app/auth.py`, `routes_auth.py`,
  `migrations/003_refresh_tokens.sql`).
- Профиль пользователя: `/auth/me` (GET/PATCH),
  загрузка/удаление аватара, раздача аватаров через presigned-URL
  (`migrations/004_user_avatar.sql`).
- Обновление схем ответа (`schemas.py`): `TokenOut` с
  `refresh_token` и `expires_in`, `MeOut` с `has_avatar`,
  `UpdateMeIn`, `RefreshIn`/`RefreshOut`.
- Автоматическое тестирование: `pytest` + `httpx`,
  фикстуры и покрытие 7 сценариев
  (`tests/conftest.py`, `tests/test_api.py`).
- Финальная сборка отчёта по ГОСТ 7.32-2017:
  расширенное содержание, оформление таблиц и иллюстраций
  (`docs/build_report.js`, `docs/report.docx`).
- Скриншоты работающего приложения в отчёте и запись
  демонстрационного видео.

**Разделы отчёта:** 2.3 (модель авторизации), 2.4 (API),
3 (практическая реализация), заключение.
